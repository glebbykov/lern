# Mini Challenges

Формат задачи:
- Вводные
- Ограничения
- Что сдать (команды + короткий разбор)

## Networking (5)
1. Подключить один контейнер к двум сетям и доказать DNS-доступность.
2. Смоделировать конфликт publish-порта и устранить его.
3. Изолировать сервисы по сетям и оставить только нужные маршруты.
4. Проверить резолв после перезапуска сервиса в compose.
5. Найти и исправить ошибку в network alias.

## Storage (5)
1. Сохранить данные в volume и подтвердить их после перезапуска.
2. Настроить backup/restore для Postgres в контейнере.
3. Показать разницу bind mount vs volume на кейсе прав доступа.
4. Добавить init-скрипт БД и проверить idempotency.
5. Смоделировать потерю данных и восстановить из резервной копии.

## Dockerfile (5)
1. Ускорить сборку перестановкой слоев.
2. Сравнить `CMD` и `ENTRYPOINT` на одном приложении.
3. Сделать multi-stage и уменьшить итоговый размер образа.
4. Убрать секрет из истории слоев.
5. Сравнить запуск приложения на slim/base и distroless.

## Security (5)
1. Перевести контейнер на non-root.
2. Запустить контейнер в read-only режиме с writable temp.
3. Применить `--cap-drop=ALL` и добавить только нужные capabilities.
4. Добавить проверку образа сканером уязвимостей.
5. Ввести правило отказа от `latest` в CI.
