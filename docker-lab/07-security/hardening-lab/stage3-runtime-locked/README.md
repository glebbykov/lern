# Этап 3 — жёсткий рантайм (правила 2, 3, 4 + лимиты)

Что добавилось в `compose.yaml`:

| Опция | Что даёт |
|---|---|
| `read_only: true` | rootfs контейнера только для чтения |
| `tmpfs: /tmp` | единственная r/w-директория, очищается при перезапуске |
| `cap_drop: [ALL]` | процесс теряет все Linux capabilities |
| `security_opt: no-new-privileges:true` | setuid-бинари не смогут эскалироваться |
| `pids_limit: 100` | защита от fork-бомб |
| `mem_limit: 128m` | OOM-kill вместо поедания памяти хоста |
| `cpus: "0.5"` | не съест больше половины ядра |

Приложение переведено на запись в `/tmp` — остальной rootfs теперь r/o,
и попытки писать в `/app` будут падать (см. stage4).

```bash
./run.sh
```
