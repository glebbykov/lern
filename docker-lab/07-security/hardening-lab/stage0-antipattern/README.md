# Этап 0 — Anti-pattern

Отправная точка, которую мы будем исправлять.

Что плохо:

1. **Базовый образ**: `ubuntu:latest` — тяжёлый, «живой» тег, куча пакетов,
   десятки известных CVE.
2. **root**: нет `USER`, PID 1 — `root`, все capabilities по умолчанию.
3. **Секрет в ENV и слоях**: `ENV SECRET=...` в Dockerfile сохраняется
   в image metadata и виден через `docker inspect`/`docker history`.
4. **Запись в rootfs**: приложение пишет `/app/requests.log`. Контейнер
   без `read_only`, поэтому любая эксплуатация может писать куда угодно.
5. **Нет лимитов**: ни памяти, ни CPU.

## Как запустить

```bash
./run.sh
```

Скрипт соберёт, поднимет и докажет все пункты выше (покажет `uid=0`,
запишет `/pwned.txt`, увидит `SECRET` в `env`).

## Как остановить

```bash
docker compose down -v
```
