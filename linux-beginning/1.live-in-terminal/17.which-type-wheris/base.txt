Введение в команды which, type, и whereis
Команды which, type, и whereis в Linux помогают находить исполняемые файлы программ, встроенные команды, а также связанные с ними страницы справки и исходные коды. Эти команды особенно полезны для администраторов и разработчиков при отладке и управлении системой.

1. Команда which
which используется для поиска исполняемых файлов программ в каталогах, указанных в переменной окружения $PATH. Она выводит полный путь к файлу программы, если она находится в одном из этих каталогов.

Пример использования:

which python
Вывод может быть таким:

/usr/bin/python
Это указывает на то, что программа python находится в каталоге /usr/bin.

Основные опции which:

-a — отображает все пути, где программа может находиться, если в системе установлено несколько версий программы.
which -a python
2. Команда type
type используется для определения того, является ли команда встроенной командой оболочки, алиасом, функцией или внешней программой. Это полезно для понимания того, как именно команда интерпретируется оболочкой.

Пример использования:

type ls
Вывод:

ls is aliased to `ls --color=auto`

Это указывает на то, что команда ls является алиасом для ls --color=auto.

Основные опции type:

-a — показывает все доступные версии команды.
type -a ls
-t — показывает только тип команды (например, "builtin", "alias" или "file").
type -t ls
3. Команда whereis
whereis находит не только исполняемые файлы программы, но и её исходные коды и страницы справки. Это полезно, если нужно узнать, где в системе расположены файлы, связанные с конкретной программой.

Пример использования:

whereis gcc
Вывод может быть таким:

gcc: /usr/bin/gcc /usr/lib/gcc /usr/share/man/man1/gcc.1.gz
Вывод показывает путь к исполняемому файлу, библиотекам и документации программы gcc.

Основные опции whereis:

-b — показывает только путь к исполняемому файлу.
whereis -b python
-m — показывает только страницы справки (man pages).
whereis -m bash
-s — показывает только исходные коды, если они есть.
whereis -s gcc
