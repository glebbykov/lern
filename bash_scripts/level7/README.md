# Level 7 — Production-ready: полный шаблон

## Что нового на этом уровне

| Приём | Зачем |
|-------|-------|
| `set -euo pipefail` + `IFS` | строгий режим с защитой от сплит-инга слов |
| `trap` на `ERR` + `EXIT` | автоочистка и отладочная информация при любом выходе |
| `VERSION` + `--version` | версионирование скрипта |
| logging с уровнями (`INFO`/`WARN`/`ERROR`/`DEBUG`) | читаемые логи в stdout и файл одновременно |
| `readonly` для констант | защита от случайной перезаписи |
| Валидация зависимостей | проверить наличие `jq`, `curl` и др. до начала работы |
| Параллельная обработка с `wait` | ускорение через фоновые процессы с отслеживанием |
| Exit codes | 0=ok, 1=ошибка, 2=неверные аргументы, 3=зависимость отсутствует |

## Скрипты

- `template.sh` — эталонный шаблон для production-скриптов (копировать как основу)
- `batch_file_processor.sh` — реальный скрипт: параллельная обработка файлов с полным набором практик

## Что значит "production-ready"

```
1. Скрипт не упадёт тихо — любая ошибка сразу видна
2. Параллельный запуск безопасен (lockfile)
3. При Ctrl+C или сигнале — очистка гарантирована
4. Логи содержат время, уровень и контекст
5. --help всегда работает
6. --dry-run позволяет проверить без риска
7. Зависимости проверяются до начала работы
8. Exit codes стандартные и документированные
```

## Чеклист перед деплоем

- [ ] `shellcheck ./script.sh` без ошибок
- [ ] Проверен `--dry-run`
- [ ] Проверен запуск без аргументов (должен показать usage)
- [ ] Проверен параллельный запуск (lockfile работает)
- [ ] Логи пишутся в файл при `-o`
- [ ] Все пути абсолютные или проверяются
