# Этап 13: Стандарт OCI и запуск через runc

В начале эпохи Docker делал всё сам: качал образы, собирал их, настраивал сеть, изоляцию и запускал процессы. 
Позже архитектуру разделили на стандарты (Open Container Initiative - OCI), чтобы любые инструменты могли работать с контейнерами одинаково.

**runc** — это низкоуровневый OCI-совместимый рантайм. Это та самая утилита, которая берёт готовую файловую систему (rootfs) и JSON-конфиг, настраивает все namespaces, cgroups, seccomp и запускает процесс. Сам Docker под капотом использует именно `runc`.

## Как запустить контейнер "вручную" через runc

### 1. Подготовка rootfs
Нам нужна папка с корневой файловой системой.
```bash
mkdir -p bundle/rootfs/bin
cp /bin/busybox bundle/rootfs/bin/
# Создаем симлинки утилит
chroot bundle/rootfs /bin/busybox --install -s /bin
```

### 2. Генерация спецификации (config.json)
Утилита `runc` может сгенерировать дефолтный шаблон конфигурации:
```bash
cd bundle
runc spec
```
Появится файл `config.json`. В нем описаны ВСЕ параметры изоляции (namespaces, cgroups, capabilities). 

### 3. Настройка и запуск
Мы можем отредактировать `config.json` (например, изменить `process.args` на `["/bin/sh", "-c", "echo Hello"]`).
Запуск контейнера:
```bash
sudo runc run my-test-container
```

В этом модуле мы собираем минимальный бандл и запускаем его исключительно силами стандарта OCI, не используя демон Docker.
