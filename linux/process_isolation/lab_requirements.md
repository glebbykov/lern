# Требования к лабораторным работам по Изоляции Процессов в Linux

Для успешного выполнения цикла лабораторных работ требуется подготовленное Linux-окружение. Все 4 работы были успешно протестированы и верифицированы. Ниже представлены системные требования и список утилит для каждого этапа.

---

## 💻 Общие системные требования

1. **Операционная система:** Любой современный дистрибутив Linux (рекомендуется Ubuntu 20.04/22.04 LTS или Debian 11/12).
2. **Ядро ОС (Kernel):** Версия **>= 4.15** (предпочтительно >= 5.8 для поддержки Cgroups v2 по умолчанию).
3. **Права доступа:** Учетная запись `root` или пользователь с доступом к полному `sudo` без пароля (для выполнения команд настройки сети, ядра и пространств имен).
4. **Init-система:** `systemd` (используется для запуска контейнеров, сессий Seccomp и управления AppArmor).

---

## 📦 Сводный список пакетов для установки
Для выполнения всего цикла работ выполните следующую команду на Ubuntu/Debian:
```bash
sudo apt-get update
sudo apt-get install -y coreutils util-linux iproute2 busybox-static python3 libcap2-bin apparmor apparmor-utils systemd-container bridge-utils curl tar
```

## 🧪 Требования по отдельным лабораторным работам

### Лабораторная работа 1: Базовая изоляция (Chroot, Namespaces)
*Цель: Ознакомление с `chroot` и базовыми командами изоляции (`unshare`)*

**Зависимости и утилиты:**
* `coreutils` — базовые утилиты (`mkdir`, `cp`, `ls`, `cat`).
* `util-linux` — предоставляет команду `unshare` для создания пространств имен.
* `libc-bin` — предоставляет утилиту `ldd` для поиска зависимостей C-библиотек (нужно для chroot).
* `iproute2` — предоставляет утилиту `ip` для управления Network namespaces (`ip netns`).
* `bash` — стандартная оболочка для изоляции.

### Лабораторная работа 2: Управление ресурсами (Cgroups v2 и Rootless)
*Цель: Ограничение CPU/RAM, создание "rootless" окружений и ручная сборка контейнера.*

**Системные требования:**
* Включенная единая иерархия **Cgroups v2** (точка монтирования `/sys/fs/cgroup/`). Можно проверить командой: `mount | grep cgroup2`.

**Зависимости и утилиты:**
* `python3` — используется для запуска скрипта, провоцирующего утечку памяти (для проверки работы OOM-Killer).
* `busybox-static` — статически слинкованная утилита, объединяющая в себе `ls`, `ps`, `id`, `sh`. Критически важна для сборки самодельного контейнера, так как не требует настройки зависимостей библиотек (`.so`) через `ldd`.

### Лабораторная работа 3: Продвинутая безопасность (Capabilities, Seccomp, AppArmor)
*Цель: Дробление прав root, фильтрация системных вызовов к ядру и мандатный контроль доступа.*

**Системные требования:**
* Ядро с поддержкой модуля `apparmor` (по умолчанию включено в Ubuntu). Можно проверить: `systemctl status apparmor`.

**Зависимости и утилиты:**
* `libcap2-bin` — предоставляет команды `setcap` и `getcap` для настройки Linux Capabilities.
* `systemd` — используется утилита `systemd-run` для запуска транзитных (временных) сервисов с наложенным фильтром Seccomp (`SystemCallFilter`).
* `apparmor` и `apparmor-utils` — предоставляют утилиту `apparmor_parser` для загрузки и выгрузки пользовательских профилей безопасности из ядра.

### Лабораторная работа 4: Виртуальные сети и systemd-nspawn
*Цель: Создание Veth-пар, мостов (Bridge) и запуск полноценных контейнеров ОС.*

**Зависимости и утилиты:**
* `bridge-utils` (или современный `iproute2`) — для создания программного коммутатора (`ip link add type bridge`).
* `systemd-container` — устанавливает утилиту `systemd-nspawn`. Это мощный встроенный инструмент systemd для развертывания легковесных Linux-контейнеров, работающий по тем же принципам, что и Docker.
* `curl` и `tar` — необходимы для скачивания и распаковки `rootfs` (минимальной корневой файловой системы) дистрибутива Alpine Linux (весит ~3 МБ).
