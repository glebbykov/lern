# Этап 14: Трассировка и безопасность с eBPF

**eBPF (Extended Berkeley Packet Filter)** — это революционная технология, которая позволяет запускать безопасные мини-программы прямо внутри ядра Linux без его перекомпиляции и без написания модулей ядра.

В контексте контейнеров eBPF используется для:
1. **Профилирования и мониторинга:** вы можете видеть ВСЕ системные вызовы контейнера, обращения к сети и диску с минимальным оверхедом (инструменты типа Tetragon, Falco, bpftrace).
2. **Сетей:** высокопроизводительная маршрутизация и балансировка пакетов (Cilium).
3. **Безопасности:** блокировка доступа к устройствам (`device cgroups v2` реализованы через eBPF) или гибкая фильтрация вызовов (альтернатива Seccomp).

В этой работе мы используем утилиту `bpftrace`, чтобы извне (с хоста) перехватить системный вызов `openat` (открытие файла), который делает процесс внутри песочницы.

## Практика: слежка за файлом

Запустим процесс, который постоянно читает файл:
```bash
while true; do cat /tmp/secret.txt; sleep 1; done &
PID=$!
```

Теперь с помощью eBPF "прикрепимся" к ядру и скажем: "Каждый раз, когда процесс с PID=$PID делает вызов sys_enter_openat, печатай имя открываемого файла":
```bash
sudo bpftrace -e 'tracepoint:syscalls:sys_enter_openat /pid == '$PID'/ { printf("Открыт файл: %s\n", str(args->filename)); }'
```

Такой подход позволяет создавать системы IDS (Intrusion Detection System), которые мгновенно обнаруживают попытки чтения `/etc/shadow` или `/run/secrets/` контейнером, не вмешиваясь в его файловую систему.
