# Лабораторная работа: Изоляция процессов в Linux

## Цель работы
Изучить базовые механизмы изоляции процессов в ОС Linux, такие как `chroot` и `namespaces` (пространства имен). Понять принципы работы контейнеризации на низком уровне.

## Теоретические сведения
Изоляция процессов в Linux строится на нескольких ключевых технологиях:
1.  **chroot** (change root) — один из старейших механизмов, позволяющий изменить корневой каталог для процесса и его потомков.
2.  **Namespaces** (Пространства имен) — механизм ядра, который позволяет изолировать системные ресурсы (PID, Network, Mount, UTS и др.).
3.  **Control Groups (cgroups)** — ограничение потребления ресурсов (CPU, RAM, I/O). *В данной работе рассматриваться не будут.*

## Часть 1: Изоляция файловой системы (chroot)

`chroot` позволяет создать "тюрьму" (jail), ограничив доступ процесса только определенной директорией.

### Практическое задание 1
1. Создайте директорию для новой корневой системы:
   ```bash
   mkdir -p /tmp/jail/{bin,lib,lib64}
   ```
2. Скопируйте оболочку `bash` и все необходимые библиотеки в jail с помощью простого скрипта:
   ```bash
   mkdir -p /tmp/jail/bin
   cp /bin/bash /tmp/jail/bin/
   
   # Автоматическое копирование всех зависимостей, которые выдает ldd
   for i in $(ldd /bin/bash | grep -o '/[a-z0-9_/\.-]*'); do
     mkdir -p "/tmp/jail$(dirname $i)"
     cp "$i" "/tmp/jail$i"
   done
   ```
   *Замечание: Команда `ldd` показывает список зависимостей (например, `libc.so.6`). Наш цикл автоматически воссоздает нужную структуру папок (например, `/lib/x86_64-linux-gnu/`) внутри "тюрьмы" и копирует туда файлы.*

3. Запустите оболочку в новой среде:
   ```bash
   chroot /tmp/jail /bin/bash
   ```
   *Попробуйте выполнить `ls`. Почему команда не найдена? Как это исправить?*

## Часть 2: UTS Namespace (Изоляция имен узлов)

UTS namespace позволяет процессу иметь собственное имя хоста (hostname) и доменное имя.

### Практическое задание 2
1. Создайте новый UTS namespace:
   ```bash
   unshare -u /bin/bash
   ```
2. Измените hostname внутри новой оболочки:
   ```bash
   hostname isolated-node
   hostname
   ```
3. Откройте второй терминал и проверьте hostname системы. Убедитесь, что он не изменился.

## Часть 3: PID Namespace (Изоляция идентификаторов процессов)

PID namespace позволяет процессу иметь собственное дерево идентификаторов, начиная с PID 1.

### Практическое задание 3
1. Запустите новый процесс в изолированном PID namespace с монтированием новой файловой системы `/proc`:
   ```bash
   unshare -p -f --mount-proc /bin/bash
   ```
2. Проверьте список процессов:
   ```bash
   ps aux
   ```
   *Вы должны увидеть, что `/bin/bash` имеет PID 1.*

## Часть 4: Network Namespace (Изоляция сети)

Позволяет изолировать сетевой стек: интерфейсы, таблицы маршрутизации, правила firewall.

### Практическое задание 4
1. Создайте сетевой namespace:
   ```bash
   ip netns add lab-ns
   ```
2. Проверьте доступные интерфейсы внутри него:
   ```bash
   ip netns exec lab-ns ip addr
   ```
3. Активируйте интерфейс-петлю (loopback):
   ```bash
   ip netns exec lab-ns ip link set lo up
   ```

1. **Сложное окружение:** Подготовьте полноценный `chroot` с базовыми командами (`ls`, `cat`, `mkdir`), используя скрипт для автоматического копирования библиотек.
2. **Связь между namespaces:** Создайте два сетевых пространства имен (`ns1`, `ns2`) и соедините их с помощью виртуального патч-корда (`veth pair`). Добейтесь того, чтобы они "пинговали" друг друга.
3. **Комбинированная изоляция:** Одной командой `unshare` создайте процесс, который одновременно изолирован по UTS, PID и Mount.
