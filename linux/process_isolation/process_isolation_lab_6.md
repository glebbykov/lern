# Лабораторная работа 6: Управление пакетами и сборка системных образов

## Цель работы
Изучить иерархию инструментов управления пакетами в Debian-подобных системах (dpkg и apt), понять структуру `.deb` пакетов и научиться создавать минимальные изолированные окружения (rootfs) с помощью `debootstrap`.

## Теоретические сведения
Управление программным обеспечением в Linux обычно делится на два уровня:
1.  **Низкоуровневый (dpkg):** Работает с отдельными файлами `.deb`. Умеет устанавливать, удалять и распаковывать их, но не умеет автоматически скачивать зависимости из интернета.
2.  **Высокоуровневый (APT - Advanced Package Tool):** Работает с репозиториями. Умеет разрешать зависимости, скачивать нужные пакеты и вызывать `dpkg` для их установки.

Пакет `.deb` — это обычный архив формата `ar`, внутри которого находятся:
*   `debian-binary`: версия формата пакета.
*   `control.tar.gz`: метаданные (название, версия, зависимости, скрипты установки).
*   `data.tar.xz` (или gz/zst): непосредственно файлы программы, которые будут разложены по системе.

## Часть 1: Анатомия .deb пакета

В этой части мы разберем пакет "на запчасти" без его установки в систему.

### Практическое задание 1
1. Скачайте пакет `hello` (простая программа, выводящая приветствие):
   ```bash
   apt-get download hello
   ```
2. Посмотрите содержимое архива с помощью утилиты `ar`:
   ```bash
   ar t hello_*.deb
   ```
3. Распакуйте содержимое пакета в текущую директорию:
   ```bash
   ar x hello_*.deb
   ```
4. Изучите состав:
   *   Распакуйте `control.tar.gz` и посмотрите файл `control` (там описаны зависимости).
   *   Распакуйте `data.tar.xz` и посмотрите, какие файлы и куда планирует установить пакет.
   ```bash
   mkdir -p pkg_data
   tar -xf data.tar.* -C pkg_data/
   find pkg_data/
   ```

## Часть 2: Ручная установка в изолированную директорию

Иногда нужно установить программу не в основную систему, а в отдельную папку (например, для создания образа контейнера).

### Практическое задание 2
1. Создайте директорию для "мини-системы":
   ```bash
   mkdir -p /tmp/mini_root
   ```
2. Распакуйте пакет в эту директорию с помощью `dpkg`:
   ```bash
   dpkg -x hello_*.deb /tmp/mini_root/
   ```
3. Попробуйте запустить программу из этой директории:
   ```bash
   /tmp/mini_root/usr/bin/hello
   ```
   *Заработает ли она? Почему? (Подсказка: проверьте зависимости через `ldd`)*

## Часть 3: Автоматизация сборки системы (debootstrap)

`debootstrap` — это стандартный инструмент для развертывания базовой системы Debian/Ubuntu в указанную директорию.

### Практическое задание 3
1. Создайте базовую систему (это может занять 1-2 минуты):
   ```bash
   # Мы используем вариант --variant=minbase для экономии места
   debootstrap --variant=minbase jammy /tmp/new_root http://archive.ubuntu.com/ubuntu/
   ```
2. Посмотрите на размер и содержимое `/tmp/new_root`. Это полноценная ОС!
3. Попробуйте войти в нее:
   ```bash
   chroot /tmp/new_root /bin/bash
   ```

## Часть 4: Оживление APT внутри chroot

Чтобы `apt` внутри новой системы мог скачивать пакеты, ему нужны виртуальные файловые системы и доступ к сети.

### Практическое задание 4
1. Смонтируйте необходимые системные ресурсы:
   ```bash
   mount -t proc proc /tmp/new_root/proc
   mount -t sysfs sys /tmp/new_root/sys
   mount --bind /dev /tmp/new_root/dev
   ```
2. Скопируйте настройки DNS, чтобы работал интернет:
   ```bash
   cp /etc/resolv.conf /tmp/new_root/etc/resolv.conf
   ```
3. Зайдите внутрь и попробуйте обновить списки пакетов:
   ```bash
   chroot /tmp/new_root /bin/bash
   # Внутри chroot:
   apt update
   apt install -y sl
   /usr/games/sl
   exit
   ```
4. **Важно:** После работы не забудьте размонтировать всё:
   ```bash
   umount /tmp/new_root/proc
   umount /tmp/new_root/sys
   umount /tmp/new_root/dev
   ```
