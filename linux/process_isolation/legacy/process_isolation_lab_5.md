# Лабораторная работа 5: Слоистые файловые системы (OverlayFS) и Copy-on-Write

## Цель работы
Понять, как устроены "образы" контейнеров (например, Docker Images) под капотом. Изучить механизм слоистых файловых систем и принципы Copy-on-Write (CoW) и Whiteout, которые позволяют мгновенно запускать сотни контейнеров из одного базового образа, не дублируя данные на диске.

## Теоретические сведения
Контейнеры (такие как Docker) не копируют операционную систему для каждого нового экземпляра. Вместо этого они используют **Union Filesystems** (чаще всего **OverlayFS** в Linux).

OverlayFS объединяет несколько директорий (слоев) в одну виртуальную:
1.  **LowerDir (Нижний слой)** — это базовый образ (например, файлы Ubuntu или Alpine). Он монтируется в режиме **Только для чтения (Read-Only)**. Их может быть несколько.
2.  **UpperDir (Верхний слой)** — пустая директория, доступная для записи. Сюда сохраняются все изменения, которые делает контейнер.
3.  **WorkDir (Рабочий каталог)** — скрытая служебная директория для атомарных операций ядра.
4.  **Merged (Объединенная директория)** — это то, что видит контейнер. В ней "склеены" нижний и верхний слои.

## Часть 1: Создание слоистой файловой системы

Соберем свой первый "образ" вручную.

### Практическое задание 1
1. Создайте структуру директорий для OverlayFS:
   ```bash
   mkdir -p /tmp/overlay/{lower,upper,work,merged}
   ```
2. Создайте файл в "базовом образе" (`lower`):
   ```bash
   echo "I am the base image" > /tmp/overlay/lower/base.txt
   ```
3. Смонтируйте OverlayFS:
   ```bash
   mount -t overlay overlay -o lowerdir=/tmp/overlay/lower,upperdir=/tmp/overlay/upper,workdir=/tmp/overlay/work /tmp/overlay/merged
   ```
4. Посмотрите содержимое `merged` (вы увидите файл из базового слоя):
   ```bash
   cat /tmp/overlay/merged/base.txt
   ```

## Часть 2: Механизм Copy-on-Write (CoW)

Если контейнер пытается изменить файл, который находится в базовом Read-Only слое, OverlayFS прозрачно копирует этот файл в `upper` слой, и только затем изменяет его. Это называется **Copy-on-Write (Копирование при записи)**.

### Практическое задание 2
1. Измените файл через точку монтирования (как если бы вы были внутри контейнера):
   ```bash
   echo "Modified in container" > /tmp/overlay/merged/base.txt
   ```
2. Загляните "под капот" — посмотрите, что реально лежит в верхнем слое (слое контейнера):
   ```bash
   cat /tmp/overlay/upper/base.txt
   ```
   *(Вы увидите измененный текст)*
3. Проверьте базовый слой:
   ```bash
   cat /tmp/overlay/lower/base.txt
   ```
   *(Текст остался оригинальным! Базовый образ не пострадал).*

## Часть 3: Механизм Whiteout (Удаление файлов)

Как контейнер может "удалить" файл, если базовый слой доступен только для чтения? Ядро Linux использует файлы-заглушки, называемые **Whiteouts**.

### Практическое задание 3
1. Удалите файл из виртуальной файловой системы:
   ```bash
   rm /tmp/overlay/merged/base.txt
   ```
2. Убедитесь, что в `merged` его больше нет:
   ```bash
   ls -l /tmp/overlay/merged/
   ```
3. Загляните в верхний слой (`upper`), чтобы понять, как ядро запомнило удаление:
   ```bash
   ls -la /tmp/overlay/upper/base.txt
   ```
   *Заметьте: файл `base.txt` в `upper` теперь является символьным устройством (`c---------`) с мажорным/минорным номерами `0, 0`. Когда процесс пытается прочитать этот файл в `merged`, OverlayFS видит устройство `0, 0` и сообщает процессу, что файла не существует (хотя физически в `lower` он лежит).*

4. Очистите за собой (размонтируйте файловую систему):
   ```bash
   umount /tmp/overlay/merged
   ```

## Самостоятельная работа: Одноразовый (Ephemeral) контейнер
Объедините знания из Лабораторных работ 1 (Namespaces) и 5 (OverlayFS).
У нас есть скачанная в Лабораторной №4 корневая система Alpine Linux в папке `/tmp/alpine_rootfs` (это будет наш базовый `lower` слой).

1. Создайте папки `upper`, `work` и `merged` в `/tmp/ephemeral/`.
2. Смонтируйте `OverlayFS`, используя Alpine как `lowerdir`.
3. Запустите `unshare -p -f -m -u -C chroot /tmp/ephemeral/merged /bin/sh`.
4. Внутри контейнера испортите систему: `rm -rf /bin /etc /lib`.
5. Выйдите из контейнера, размонтируйте OverlayFS и очистите `upper` и `work`.
6. Снова смонтируйте OverlayFS и запустите контейнер — система будет как новая, потому что оригинальный `lower` слой не пострадал!
