# 09 — Сеть: контрольные вопросы

**Чем veth отличается от bridge?**
veth — это L2-«патч-корд» из двух интерфейсов. То что вошло в один,
вышло из другого, и всё. Bridge — программный switch, объединяет
несколько L2-портов в одну широковещательную сеть. Контейнеры
подключаются к bridge через veth-пары: один конец в netns контейнера,
второй в bridge на хосте.

**Зачем мосту IP-адрес?**
Если ты хочешь, чтобы контейнеры могли общаться с хостом или ходить
во внешний мир через хост — мост должен быть L3-устройством с IP.
Этот IP служит default gateway внутри контейнера. Без IP мост — чисто
L2, контейнеры друг друга видят, хост — нет.

**Что такое MASQUERADE и чем отличается от SNAT?**
MASQUERADE = SNAT, у которого source IP берётся динамически от того
интерфейса, через который пакет уходит. Удобно когда у хоста IP
меняется (DHCP). SNAT нужно прописывать явно: `--to-source 1.2.3.4`.

**Почему нужен `net.ipv4.ip_forward=1`?**
По умолчанию Linux не пересылает пакеты с одного интерфейса на другой
(ведёт себя как хост, не роутер). `ip_forward=1` превращает его в
роутер — без этого пакеты из контейнера не уйдут наружу.

**Как работает Docker `--network host`?**
Контейнер не получает свой netns — использует netns хоста. То есть
`localhost:8080` внутри контейнера = `localhost:8080` на хосте.
Меньше изоляции, но нет оверхеда виртуальной сети.
