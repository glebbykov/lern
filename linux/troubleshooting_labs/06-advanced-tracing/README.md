# Урок 6: Продвинутая трассировка (strace)

## Цель
Понять, почему процесс "завис" или не может запуститься должным образом, если в логах ничего нет.

## Основные команды
- `strace <команда>` — запустить команду и вывести все её обращения к ядру (системные вызовы).
- `strace -p <PID>` — прикрепиться к уже работающему процессу и смотреть, что он делает прямо сейчас.
- `strace -e openat -p <PID>` — фильтровать вывод, показывая только вызовы открытия файлов.
- `ltrace -p <PID>` — похожа на strace, но показывает вызовы библиотечных функций (например, `malloc`, `printf`).
- `nice` / `renice` — изменение приоритета процесса, если он мешает работать системе.

## Задание
1. Запустите `./simulate.sh`. Скрипт запустит фоновый процесс Python, который ничего не пишет в логи, но почему-то не работает как нужно.
2. Найдите PID этого процесса (например, `ps aux | grep stuck_app`).
3. Прикрепитесь к нему с помощью `strace -p <PID>`.
4. Вы увидите, что процесс в бесконечном цикле пытается открыть определенный конфигурационный файл и получает ошибку `ENOENT (No such file or directory)`.
5. Создайте этот недостающий файл (`touch /tmp/missing_config.conf`).
6. Снова посмотрите в `strace` — вызов должен стать успешным, а скрипт завершит свою работу!
