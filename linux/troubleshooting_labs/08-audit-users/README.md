# Урок 8: Аудит и Активность пользователей

## Цель
Узнать, кто изменяет критически важные файлы, и найти следы подозрительной активности пользователей.

## Основные команды
- `last` — показывает историю успешных входов в систему (читает `/var/log/wtmp`).
- `lastb` — показывает неудачные попытки входа (читает `/var/log/btmp`). Отличный инструмент для обнаружения брутфорса SSH!
- `auditctl -w <путь> -p <права> -k <ключ>` — установить правило аудита (наблюдения) за файлом (требует установленного `auditd`).
  - Пример: `auditctl -w /etc/shadow -p wa -k shadow_changes` (следить за записью/атрибутами, ключ поиска `shadow_changes`).
- `ausearch -k <ключ>` — найти события аудита по ключу.
- `logger "Сообщение"` — вручную отправить запись в системный лог (`syslog` / `journald`).

## Задание
1. Запустите `./simulate.sh`.
2. Скрипт настроит правило `auditd` на наблюдение за важным файлом `/tmp/top_secret.txt` и затем сымитирует его изменение другим пользователем.
3. Кроме того, скрипт сгенерирует несколько "ошибок" неудачного входа в SSH.
4. Ваша задача:
   - Использовать `lastb`, чтобы увидеть список неудачных логинов (какой пользователь пытался войти?).
   - Использовать `ausearch -k secret_watch`, чтобы найти событие изменения файла (посмотрите поле `uid=` или `exe=`, чтобы понять, кто или какая программа его изменила).
