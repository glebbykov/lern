# Урок 5: Базовая работа с Сертификатами (SSL/TLS)

## Цель
Научиться диагностировать проблемы с HTTPS и сертификатами.

## Основные команды
- `openssl x509 -in cert.crt -text -noout` — прочитать содержимое сертификата в человекочитаемом виде (кто выдал, кому, до какого числа годен).
- `openssl x509 -enddate -noout -in cert.crt` — быстро проверить дату истечения сертификата.
- `openssl rsa -check -in private.key` — проверить целостность приватного ключа.
- `openssl x509 -noout -modulus -in cert.crt | openssl md5` и `openssl rsa -noout -modulus -in private.key | openssl md5` — **Важнейший трюк!** Если хэши совпадают, значит этот ключ 100% от этого сертификата.
- `curl -vI https://domain.com` — проверить SSL-рукопожатие при запросе (покажет, если сертификат просрочен или не доверяется).
- `openssl s_client -connect domain.com:443` — дебаг TLS-соединения (какие сертификаты сервер отдаёт при подключении).

## Задание
1. Запустите `./simulate.sh`. Скрипт сгенерирует 2 сертификата и 2 ключа в папке `/tmp/certs/`.
2. Один сертификат просрочен, второй — нормальный. Один ключ подходит к первому, другой ко второму. Всё перепутано!
3. Ваша задача с помощью команд `openssl x509` и `openssl rsa`:
   - Найти, какой из сертификатов (`certA.crt` или `certB.crt`) ещё действителен (valid).
   - Найти, какой из приватных ключей (`key1.key` или `key2.key`) соответствует валидному сертификату (через сравнение modulus).
