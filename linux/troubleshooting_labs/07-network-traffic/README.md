# Урок 7: Анализ сетевого трафика (tcpdump)

## Цель
Узнать, куда машина отправляет сетевые запросы, когда мы не знаем, какой именно процесс это делает.

## Основные команды
- `tcpdump -i any` — слушать трафик на всех интерфейсах.
- `tcpdump -i any port 80 -n` — слушать только HTTP-трафик, не резолвить IP в домены (чтобы было быстрее).
- `tcpdump -i any host 8.8.8.8` — слушать трафик только до конкретного хоста.
- `tcpdump -i eth0 -w traffic.pcap` — записать трафик в файл для последующего анализа в Wireshark.
- `iftop` / `vnstat` / `nload` — утилиты для визуального мониторинга пропускной способности (какие IP потребляют канал).

## Задание
1. Запустите `./simulate.sh`. В фоне запустится процесс, который имитирует "утечку" данных — постоянно шлет HTTP-запросы на определенный внешний IP.
2. Используйте `tcpdump -i any port 80 -n`, чтобы перехватить эти пакеты.
3. Посмотрите на вывод tcpdump и найдите, на какой целевой IP-адрес (Destination IP) постоянно идут запросы.
4. Выяснив IP, убейте фоновый процесс (`killall curl` или найдите скрипт-генератор в `ps aux`).
