# Урок 2: Диски и Ввод/Вывод (I/O)

## Цель
Найти, куда пропало место на диске.

## Основные команды
- `df -h` — показать свободное место на всех примонтированных дисках.
- `df -i` — показать свободные Inode (иногда место есть, а файлы создавать нельзя, т.к. кончились Inode).
- `du -sh /* 2>/dev/null` — оценить размер папок в корне.
- `ncdu /` — удобная интерактивная утилита для поиска больших файлов (ncurses du).
- `lsof +L1` — **Важнейшая команда!** Показывает файлы, которые были удалены (`deleted`), но их размер всё ещё занимает место на диске, потому что какой-то процесс удерживает их открытыми.
- `lsblk` — список блочных устройств (дисков).
- `iostat -xz 1` — посмотреть нагрузку на диски (I/O) в реальном времени.

## Задание
1. Запустите `./simulate.sh`.
2. Скрипт создаст большой файл, а затем "удалит" его, но продолжит держать открытым. Место на диске уменьшится!
3. Используйте `df -h`, чтобы увидеть нехватку места (например, в `/tmp`).
4. Используйте `lsof +L1`, чтобы найти "удаленный", но удерживаемый процессом файл.
5. Убейте процесс, который его держит (`kill -9 <PID>`), и проверьте `df -h` снова — место должно вернуться.
