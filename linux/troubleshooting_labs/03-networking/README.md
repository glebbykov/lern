# Урок 3: Сеть и Порты

## Цель
Понять, почему сетевой сервис недоступен, хотя он "запущен".

## Основные команды
- `ip a` — посмотреть свои IP-адреса.
- `ss -tulnp` — показать, какие процессы слушают (Listen) какие TCP/UDP порты.
- `ping <host>` — проверить базовую L3-доступность.
- `nc -vz <host> <port>` — проверить, открыт ли конкретный TCP-порт (например, `nc -vz 127.0.0.1 8080`).
- `curl -I <url>` — сделать HTTP HEAD запрос (проверка веб-сервера).
- `iptables -L -n -v` — посмотреть правила фаервола (возможно, порт заблокирован).

## Задание
1. Запустите `./simulate.sh`. Он запустит веб-сервер на порту 8888.
2. Попробуйте сделать `curl http://127.0.0.1:8888`. Запрос "зависнет".
3. С помощью `ss -tulnp | grep 8888` убедитесь, что сервер реально работает и слушает порт.
4. Проверьте фаервол: `iptables -L -n -v`. Вы увидите правило DROP для порта 8888.
5. Удалите правило (например, `iptables -F` для очистки всех правил, или `iptables -D INPUT ...`) и убедитесь, что `curl` заработал.
