
# Лабораторная работа 4: Подпись в Git

## Цель
Научиться использовать систему подписи в Git для обеспечения безопасности коммитов и тегов.

## Требования
- Установленный GPG и настроенный персональный ключ
- Репозиторий Git для работы

## 1. Введение в GPG

Во-первых, необходимо убедиться, что у вас есть настроенный GPG и персональный ключ. Для проверки ключей используйте команду:

```bash
$ gpg --list-keys
```

Если ключ отсутствует, сгенерируйте его с помощью команды:

```bash
$ gpg --gen-key
```

## 2. Настройка GPG в Git

После генерации ключа, настройте Git для использования этого ключа:

```bash
$ git config --global user.signingkey <ID ключа>
```

## 3. Подпись тегов

Подпишите новый тег, используя команду:

```bash
$ git tag -s v1.0 -m 'Подписанный тег версии 1.0'
```

Затем проверьте информацию о теге:

```bash
$ git show v1.0
```

## 4. Проверка тегов

Для проверки подписанного тега используйте команду:

```bash
$ git tag -v v1.0
```

Эта команда проверит подпись с помощью GPG и выведет результат.

## 5. Подпись коммитов

Вы также можете подписывать коммиты:

```bash
$ git commit -S -m 'Подписанный коммит'
```

Для просмотра подписей коммитов используйте:

```bash
$ git log --show-signature
```

## 6. Проверка всех подписей

Для проверки всех подписей коммитов выполните:

```bash
$ git log --pretty="format:%h %G? %aN  %s"
```

Символы, отображаемые в поле `%G?`, означают:
- `G` - подпись действительна
- `N` - коммит не подписан

## 7. Подпись при слиянии

При слиянии можно проверить подписи и подписать результат:

```bash
$ git merge --verify-signatures -S signed-branch
```

## Заключение

Использование подписей помогает повысить безопасность вашего кода и доверие к коммитам. Убедитесь, что все члены вашей команды знакомы с процессом и понимают его важность.
